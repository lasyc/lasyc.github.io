Java Web

Java Web 是 Java 语言在 Web 上的应用。客户端通过发送请求到指定的服务器，服务器在接受请求之后调用对应的处理器，处理请求，返回结果给客户端。

Web 资源的分类

  1. 静态资源：HTML、CSS、JavaScript、图片、视频、音频、字体文件、图标、插件等等
  2. 动态资源：Servlet、JSP、PHP、ASP、CGI 等

常用的 Web 容器

  1. Tomcat：Apache 官方提供的 Web 容器
  2. JBoss：Red Hat 官方提供的 Web 容器
  3. Glassfish：Oracle 官方提供的 Web 容器
  4. Resin：Caucho 官方提供的 Web 容器
  5. WebLogic：Oracle 官方提供的 Web 容器

下载安装

下载并解压即可

目录

  1. bin: 存放 Tomcat 的可执行程序
  2. conf: 存放 Tomcat 的配置文件
  3. lib: 存放 Tomcat 的依赖库
  4. logs: 存放 Tomcat 的日志文件
  5. webapps: 存放 Tomcat 的 Web 应用程序
  6. work: 存放 Tomcat 的临时文件

启动

  1. 双击 Tomcat 安装路径 bin 文件夹下的 startup.bat
  2. 通过命令行进入 bin 文件夹下，执行命令 catalina run（推荐）

停止

  1. 关闭弹窗
  2. Ctrl + C
  3. 双击 Tomcat 安装路径 bin 文件夹下的 shutdown.bat（推荐）

修改配置

Tomcat 的一切配置都在 conf 文件夹下，修改任何配置都需要重启 Tomcat

  1. 修改端口号：server.xml 中的 Connector 标签的 port 属性

部署

  1. 直接将 html 文件放到 Tomcat 的 webapps 文件夹下即可，无需打包
  2. 在 Tomcat 的 conf/Catalina/localhost 文件夹下，创建一个 xxxx.xml 文件，文件内容 <Context path="访问路径" docBase="工程路径" />，这样 html 文件就可以放到任意位置了

区别

直接打开 HTML 文件和通过 Tomcat 访问 HTML 文件的区别是：

  1. 直接打开 HTML 文件，是本地文件协议，因此地址栏为：file:///本地磁盘路径/index.html
  2. 通过 Tomcat 访问 HTML 文件，是超文本传输协议，因此地址栏为：http://localhost:8080/访问路径/index.html
  3. 通过 file:/// 访问文件，会受到浏览器的安全限制，不能访问其他域名的资源，不能使用某些 API

ROOT 工程

ROOT 工程不需要在浏览器地址栏中添加访问路径，直接通过 http://localhost:8080/ 即可访问，且默认访问 index.html 文件，其次访问 index.jsp 文件等

IDEA 集成 Tomcat

  1. settings（设置） -> Build, Execution, Deployment（构建，执行，部署） -> Application Servers（应用程序服务器）
  2. 点击 + 按钮，选择 Tomcat
  3. 输入 Tomcat 安装路径，点击 OK 按钮

IDEA 创建动态工程

旧版本 IDEA

  1. 选择 Java Enterprise
  2. 选择 Tomcat 服务器
  3. 勾选 Web Application 选项
  4. 勾选 Create web.xml
  5. 点击 Next 按钮

新版本 IDEA

  1. 新建 Java 工程
  2. 完成后点击工程名，双击 shift 键，选择 Add Framework Support（添加框架支持）
  3. 勾选 Web Application 选项，点击确定
  4. 点击右上角的 当前文件 下拉列表，选择 编辑配置
  5. 点击左上角 + 号，添加本地 Tomcat 服务器。

文件结构

在 web/WEB-INF 目录下，创建一个 lib 文件夹，用于存放依赖的 jar 包

  1. src：Java 源代码目录
  2. web：资源文件，如 HTML、CSS、JavaScript 等
  3. WEB-INF：受保护的文件夹，浏览器无法直接访问
  4. web.xml：Web 应用配置文件，用于配置 Web 应用的初始化参数、过滤器、监听器等
  5. lib：存放依赖的 jar 包

添加依赖

右键 lib 文件夹，选择 Add as Library（添加为库）
