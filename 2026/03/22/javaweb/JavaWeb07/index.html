Servlet

  1. Servlet 是 Java Web 中的一种技术，用于处理客户端的请求和响应
  2. Servlet 是 JavaWeb 三大组件之一，其他两个是 Filter（过滤器）和 Listener（监听器）

第一个 Servlet 程序

Servlet 是一个接口，因此我们需要实现 Servlet 接口，才能创建一个 Servlet 程序

注意：Servlet 接口在 JDK8 之后就不存在 javax.servlet.Servlet 中了，需要使用 jakarta.servlet.Servlet，或者修改 JDK 到 8 以下版本

jakarta.servlet-api

  1. 创建一个类，实现 Servlet 接口
  2. 重写 service 方法，用于处理客户端的请求和响应
  3. 编辑 web.xml 文件，将 Servlet 类名和访问路径映射起来

当浏览器访问 http://localhost:8080/工程名/api/test 时，会调用 com.codedog.servlet 包下 TestServlet 类的 service 方法

Servlet 生命周期

  1. 执行 Servlet 构造器方法
  2. 执行 init 方法
  3. 执行 service 方法
  4. 执行 destroy 方法

当启动 Tomcat 服务器后，Servlet 并不会被创建，直到浏览器访问该 Servlet 才会被创建，会先执行 构造器方法，再执行 init 方法，且只会执行一次，当每次接收到请求时，会执行 service 方法，可以多次执行，当 Tomcat 服务器关闭时，会执行 destroy 方法，

请求分发

在前边学习 Form 表单的时候我们学习过，请求方法分为 GET 和 POST 两种，而在 Servlet 中，我们只能通过 service 方法来处理请求和响应，因此，我们需要在 service 方法中判断请求方法，根据不同的请求方法，调用不同的方法来处理请求和响应

在此基础上，我们可以将 GET 方法和 POST 方法分别处理，定义 doGet 方法和 doPost 方法，在 service 方法中判断请求方法，根据不同的请求方法，调用不同的方法来处理请求和响应

简化 Servlet 的开发

前边学习了使用实现类来实现 Servlet 接口的方式创建 Servlet，除此之外，我们还可以直接继承 Servelt 的实现类，如：HttpServlet，可以简化 Servlet 的开发，是一般工程中常用的方法

使用 IDEA 生成 Servlet

  1. 右键包 -> New -> Create New Servlet
  2. 输入Servlet 别名，Servelt 所在包名，Servlet 全类名
  3. 点击 OK 按钮

Servket 继承体系

ServlecConfig 类

ServletConfig 类是一个接口，用于获取 Servlet 的配置信息，如：Servlet 的别名、初始化参数等，每一个 Servlet 都有一个 ServletConfig 对象，用于获取 Servlet 的配置信息

ServletConfig 被保存在 GenericServlet 类中，在 Servlet 中可以通过 getServletConfig() 方法获取，是 GenericServlet 在初始化时通过 init 方法保存的，因此，当我们通过继承创建 Servlet 时，如果我们重写 init 方法，必须调用 super.init(servletConfig) 方法，来确保 ServletConfig 被保存到 GenericServlet 中，否在后续我们使用 getServletConfig() 获取 servletConfig 对象会返回 null

ServletContext 类

ServletContext 类是一个接口，用于获取 Servlet 的上下文信息，且一个工程只有一个 ServletContext 对象，通过 ServletConfig 调用 getServletContext() 方法获取

用途

  1. 获取 web.xml 文件中的初始化参数
  2. 获取工程路径
  3. 获取工程在服务器上的绝对路径
  4. 可以向 Map 一样通过 setXXX 和 getXXX 方法，保存和获取数据

HTTP 协议

什么是 HTTP

HTTP（Hyper Text Transfer Protocol，超文本传输协议）是互联网上应用最广泛的协议，是万维网（WWW）的基础，也是 World Wide Web（W3C）的 abbreviation。

HTTP 是一个请求-响应协议，即浏览器向服务器请求数据，服务器返回数据给浏览器。

HTTP 协议格式

根据不同的请求方式，协议格式如下：

GET 请求

请求行

  1. 请求方式：GET
  2. 请求资源路径：/servlet_study/api/context
  3. HTTP 协议版本：HTTP/1.1

请求头

  1. Accept：表示浏览器可以接收的 MIME 类型
  2. Accept-Encoding：表示浏览器可以接收的压缩编码
  3. Accept-Language：表示浏览器可以接收的语言
  4. Connection：表示浏览器是否保持连接
  5. Cookie：表示浏览器发送给服务器的 Cookie
  6. Host：表示请求的域名和端口号
  7. Sec-Fetch-Dest：表示浏览器的请求目的
  8. Sec-Fetch-Mode：表示浏览器的请求模式
  9. Sec-Fetch-Site：表示浏览器的请求来源
  10. Sec-Fetch-User：表示浏览器的请求用户
  11. Upgrade-Insecure-Requests：表示浏览器是否允许不安全的请求
  12. User-Agent：表示浏览器信息
  13. sec-ch-ua：表示浏览器的 MIME 类型
  14. sec-ch-ua-mobile：表示浏览器的移动设备
  15. sec-ch-ua-platform：表示浏览器的平台

POST 请求

请求行

  1. 请求方式：POST
  2. 请求资源路径：/servlet_study/api/context
  3. HTTP 协议版本：HTTP/1.1

请求头：相比与 GET 请求，多了一个空行

  1. Cache-Control：表示缓存控制
  2. Content-Length：表示请求体的长度
  3. Content-Type：表示请求体的 MIME 类型
  4. Origin：表示请求的来源
  5. Referer：表示请求的来源

请求体：发送给服务器的数据

常用请求头

  1. Accept：表示浏览器可以接收的 MIME 类型
  2. Accept-Encoding：表示浏览器可以接收的压缩编码
  3. Accept-Language：表示浏览器可以接收的语言
  4. User-Agent：表示浏览器信息
  5. Host：表示请求的域名和端口号
  6. Referer：表示请求的来源
  7. Cookie：表示浏览器发送给服务器的 Cookie

HTTP 响应格式

响应行

  1. HTTP 协议版本：HTTP/1.1
  2. 状态码：表示响应的状态码

响应头

  1. Content-Type：表示响应体的 MIME 类型
  2. Content-Length：表示响应体的长度
  3. Date：表示响应的时间
  4. Keep-Alive：表示浏览器是否保持连接
  5. Connection：表示浏览器是否保持连接

响应体

状态码

  1. 200：表示请求成功
  2. 404：表示请求的资源不存在
  3. 500：表示服务器内部错误
  4. 302：表示重定向
  5. 403：表示请求被禁止
  6. 400：表示请求错误

MIME 类型

MIME（Multipurpose Internet Mail Extensions，多用途因特网邮件扩展）是一种用于描述互联网资源类型的标准。

HttpServletRequest 类

HttpServletRequest 类封装了客户端的请求，提供了一系列的方法来获取请求参数、请求头、请求体、请求方法、请求路径、请求参数等信息。

方法 描述
Cookie[] getCookies() 返回一个数组，包含客户端发送该请求的所有的 Cookie 对象。
Enumeration getAttributeNames() 返回一个枚举，包含提供给该请求可用的属性名称。
Enumeration getHeaderNames() 返回一个枚举，包含在该请求中包含的所有的头名。
Enumeration getParameterNames() 返回一个 String 对象的枚举，包含在该请求中包含的参数的名称。
HttpSession getSession() 返回与该请求关联的当前 session 会话，或者如果请求没有 session 会话，则创建一个。
HttpSession getSession(boolean create) 返回与该请求关联的当前 HttpSession，或者如果没有当前会话，且创建是真的，则返回一个新的 session 会话。
Locale getLocale() 基于 Accept-Language 头，返回客户端接受内容的首选的区域设置。
Object getAttribute(String name) 以对象形式返回已命名属性的值，如果没有给定名称的属性存在，则返回 null。
ServletInputStream getInputStream() 使用 ServletInputStream，以二进制数据形式检索请求的主体。
String getAuthType() 返回用于保护 Servlet 的身份验证方案的名称，例如，”BASIC” 或 “SSL”，如果JSP没有受到保护则返回 null。
String getCharacterEncoding() 返回请求主体中使用的字符编码的名称。
String getContentType() 返回请求主体的 MIME 类型，如果不知道类型则返回 null。
String getContextPath() 返回指示请求上下文的请求 URI 部分。
String getHeader(String name) 以字符串形式返回指定的请求头的值。
String getMethod() 返回请求的 HTTP 方法的名称，例如，GET、POST 或 PUT。
String getParameter(String name) 以字符串形式返回请求参数的值，或者如果参数不存在则返回 null。
String getPathInfo() 当请求发出时，返回与客户端发送的 URL 相关的任何额外的路径信息。
String getProtocol() 返回请求协议的名称和版本。
String getQueryString() 返回包含在路径后的请求 URL 中的查询字符串。
String getRemoteAddr() 返回发送请求的客户端的互联网协议（IP）地址。
String getRemoteHost() 返回发送请求的客户端的完全限定名称。
String getRemoteUser() 如果用户已通过身份验证，则返回发出请求的登录用户，或者如果用户未通过身份验证，则返回 null。

String getRequestedSessionId() 返回由客户端指定的 session 会话 ID。
String getServletPath() 返回调用 JSP 的请求的 URL 的一部分。
String[] getParameterValues(String name) 返回一个字符串对象的数组，包含所有给定的请求参数的值，如果参数不存在则返回 null。
boolean isSecure() 返回一个布尔值，指示请求是否使用安全通道，如 HTTPS。
int getContentLength() 以字节为单位返回请求主体的长度，并提供输入流，或者如果长度未知则返回 -1。
int getIntHeader(String name) 返回指定的请求头的值为一个 int 值。
int getServerPort() 返回接收到这个请求的端口号。
getParameterMap() 将参数封装成 Map 类型。

请求转发

请求转发 是指服务器在响应客户端请求时，将请求转发给其他资源，然后返回给客户端。

使用 getRequestDispatcher() 方法获取 RequestDispatcher 对象，然后调用 forward() 方法。

  1. ULR 会显示请求的 URL，被视为一次请求
  2. 重定向后可以取得请求参数
  3. 共享请求参数

base 标签

当我们使用相对路径进行页面跳转时，如果参照的路径发生了变化，那么就会导致相对路径错误，以至于找不到页面。

我们可以使用 base 标签来解决这个问题，base 标签可以设置一个默认的 URL，当使用相对路径时，会自动加上这个默认的 URL。

  1. 采用直接跳转的方式，URL 为 http://localhost:8080/servlet_study/a/b/c/c.html，因此通过相对路径可以的到转换后的路径是 http://localhost:8080/servlet_study/index.jsp
  2. 但是采用重定向跳转的方式，URL 为 http://localhost:8080/servlet_study/api/servlet1，因此通过相对路径可以的到转换后的路径是 http://localhost:8080/index.jsp，这才导致无法找到页面

路径

  1. 相对路径：从当前页面出发，按照路径找到目标页面
  2. 绝对路径：从根目录出发，按照路径找到目标页面

在 JavaWeb 工程中，在静态页面中，/ 会被解析成 http://localhost:8080/，而在服务器中，/ 会被解析成 http://localhost:8080/servlet_study/

HttpServletResponse 类

HttpServletResponse 类用于向客户端发送响应，可以通过一些列方法来设置响应头、响应体、响应状态码等信息。

头信息 描述
Allow 这个头信息指定服务器支持的请求方法（GET、POST 等）。
Cache-Control 这个头信息指定响应文档在何种情况下可以安全地缓存。可能的值有：public、private 或 no-cache 等。Public 意味着文档是可缓存，Private 意味着文档是单个用户私用文档，且只能存储在私有（非共享）缓存中，no-cache 意味着文档不应被缓存。
Connection 这个头信息指示浏览器是否使用持久 HTTP 连接。值 close 指示浏览器不使用持久 HTTP 连接，值 keep-alive 意味着使用持久连接。
Content-Disposition 这个头信息可以让您请求浏览器要求用户以给定名称的文件把响应保存到磁盘。
Content-Encoding 在传输过程中，这个头信息指定页面的编码方式。
Content-Language 这个头信息表示文档编写所使用的语言。例如，en、en-us、ru 等。
Content-Length 这个头信息指示响应中的字节数。只有当浏览器使用持久（keep-alive）HTTP 连接时才需要这些信息。
Content-Type 这个头信息提供了响应文档的 MIME（Multipurpose Internet Mail Extension）类型。
Expires 这个头信息指定内容过期的时间，在这之后内容不再被缓存。
Last-Modified 这个头信息指示文档的最后修改时间。然后，客户端可以缓存文件，并在以后的请求中通过 If-Modified-Since 请求头信息提供一个日期。
Location 这个头信息应被包含在所有的带有状态码的响应中。在 300s 内，这会通知浏览器文档的地址。浏览器会自动重新连接到这个位置，并获取新的文档。
Refresh 这个头信息指定浏览器应该如何尽快请求更新的页面。您可以指定页面刷新的秒数。
Retry-After 这个头信息可以与 503（Service Unavailable 服务不可用）响应配合使用，这会告诉客户端多久就可以重复它的请求。
Set-Cookie 这个头信息指定一个与页面关联的 cookie。

响应流

如果想要向客户端发送数据，则需要使用流，分为：字节流和字符流

  1. 字节流：getOutputStream()，用于文件下载等二进制数据
  2. 字符流：getWriter()，用于字符串或文本信息

请求重定向

请求重定向是指服务器在接收请求后，由于种种原因无法提供服务，可以使用请求重定向将新的请求路径反馈给客户端，客户端在接收到新的请求地址后再次发送请求

  1. ULR 会显示更新重定向后的 URL，被视为两次请求
  2. 重定向后无法取得请求参数
  3. 可以自定义状态码
  4. 可以重定向到外部网站

方式一

使用 setStatus() 和 setHeader() 方法，可以手动控制跳转。

方式二

使用 sendRedirect() 方法。

三层架构

JavaEE 三层架构分别是：

  1. 视图展示层：获取请求参数，封装数据，调用数据访问层，返回结果给视图展示层
  2. 业务逻辑层：获取业务参数，调用数据访问层，返回结果给业务逻辑层
  3. 数据访问层：获取数据访问参数，调用数据库，返回结果给数据访问层
