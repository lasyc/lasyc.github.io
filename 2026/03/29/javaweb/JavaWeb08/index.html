JSP

JSP 是 Java Server Page 的简称，是一种动态网页技术。

早期的 HTML 页面只能静态显示数据，因此想要更新数据，就只能再后端通过字符串绘制 HTML 页面。然后传给前端，前端再进行展示。

JSP 允许在 HTML 中嵌入 Java 代码，从而实现动态网页。从而让 HTML 中的数据实现动态更新的效果。

JSP 本质

JSP 本质是一个 Java 类，通过编译成 .class 文件，再运行。该类继承了 HttpJspBase 类，而 HttpJspBase 类继承了 HttpServlet 类，所以 JSP 本质是 HttpServlet 类的一个子类。再该类的内部其实也是通过 resp 将 HTML 拼接成字符串返回给前端。只不过不用我们自己拼接字符串了，JSP 会自动完成拼接。

语法

Page 指令

Page 指令用于指定 JSP 页面的属性。

  1. language: 指定 JSP 页面的脚本语言。
  2. contentType: 指定 JSP 页面的内容类型。
  3. pageEncoding: 指定 JSP 页面的编码。
  4. import: 指定导入的类。
  5. autoFlush: 指定是否自动刷新。
  6. buffer: 指定缓冲区大小。
  7. errorPage: 错误页面。
  8. isErrorPage: 是否是错误页面。
  9. session: 是否使用 session。
  10. extends: 指定继承的父类。

注释

在 JSP 中，有三种的注释方式：

  1. HTML 注释：<!-- 注释内容 -->
  2. JSP 注释：<%-- 注释内容 --%>
  3. Java 注释：<% /* 注释内容 */ %>

内置对象

因为 JSP 本质就是一个 Servlet，所以 JSP 中内置了和 Servlet 一样的对象。

对象 描述
request HttpServletRequest类的实例，代表 HTTP 请求的对象，包含客户端发送到服务器的信息，如表单数据、URL参数等。
response HttpServletResponse类的实例，代表 HTTP 响应的对象，用于向客户端发送数据和响应。
out JspWriter类的实例，用于向客户端输出文本内容的对象，通常用于生成HTML。
session HttpSession类的实例，代表用户会话的对象，可用于存储和检索用户特定的数据，跨多个页面。
application ServletContext类的实例，代表 Web 应用程序的上下文，可以用于存储和检索全局应用程序数据。
config ServletConfig类的实例，包含有关当前 JSP 页面的配置信息，例如初始化参数。
pageContext PageContext类的实例，提供对JSP页面所有对象以及命名空间的访问
page 类似于 Java 类中的 this 关键字，代表当前 JSP 页面的实例，可以用于调用页面的方法。
exception exception 类的对象，代表发生错误的 JSP 页面中对应的异常对象，用于处理 JSP 页面中的异常情况，可用于捕获和处理页面中发生的异常。需要在 JSP 中开启 isErrorPage="true"

域对象

在上述九个对象中，有四个域对象：request、session、application、pageContext。

这四个与对象，其中都提供了 getXXX 和 setXXX 方法，可以像 Map 一样存取数据。

作用范围：

对象 作用范围
request 一次请求，发送请求到接收响应
session 一次会话，打开浏览器直至关闭
application 一次应用，项目启动直至关闭
pageContext 一次页面，当前 JSP 页面

范围从小到大依次是：pageContext > request > session > application

out 和 response.getWriter()

out 和 response.getWriter() 都是 JSP 页面中的对象，用于向客户端输出内容。当我们同时使用位置，out 和 response.getWriter() 输出内容时，out 输出的内容会追加到 response.getWriter() 输出的内容之后。

out 其实也是通过 ressponse.getWriter() 获取的，但是与 response.getWriter() 的区别是 out 是 JSP 默认对象，response.getWriter() 是 JSP 创建的对象。并不是同一个对象。

在 JSP 文件中，默认使用 out 来输出内容。因此当我们在 JSP 页面想要输出内容时，可以使用 out 对象。尽可能与 JSP 相同，避免页面结构错乱。

在 out 对象中，存在两个方法：write() 和 print()，在输出字符串时，两者没有任何区别，但是当出书数值时，write() 会将数值通过编码转换成字符，而 print() 会将数值转换成字符串。

动态包含

动态包含所实现的效果同样，只是动态包含的实现方式不同，需要使用<jsp:include page="xxx.jsp" />标签来实现。

静态包含会将公共文件和主体文件作为一个整体，只会生成一个 servlet 文件，而动态包含则会生成多个 servlet 文件。会将公共文件单独生成 servlet 文件，然后通过 JspRuntime.include(request, response, page) 方法将公共文件包含到主体文件中。

动态包含的底层调用了 JspRuntime.include(request, response, page) 将自己的 request、response、page 传递给公共文件，因此当公共文件通过 out 输出时，其实用的是主体文件的，因此才能将内容都整合到一起显示到页面上。

因此，使用动态包含，还可以通过 <jsp:param name="xxx" value="xxx" /> 向公共文件中传递参数。

Listener 监听器

Listener 监听器，同 Servlet、Filter 过滤器一样，是 Java Web 架构中的三大组件之一。

Listener 监听器就是用于监听某个事件，当事件发生时，会触发监听器中的方法。

监听器分类

监听器有很多的分类，但是目前常用的只有 ServletContextListener，可以用于监听 ServletContext 的创建和销毁事件。
