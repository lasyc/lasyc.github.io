Session

Session 就是会话，一个 Session 负责维护一个客户端，每个客户端都有自己的 Session 会话

创建 Session

每个 Session 都有一个属于自己的唯一 ID

  • request.getSession()：获取当前连接的 Session 对象，如果不存在则会自动创建
  • session.isNew()：判断当前 Session 是自动创建的，还是获取之前的
  • session.getId()：获取当前 Session 的唯一 ID

向 Session 域中保存数据

Session 的存活时间

  • setMaxInactiveInterval: 设置 Session 的超时时间，超过时间，Session 会被销毁，值为正数表示指定时间后销毁，负数表示永不过期
  • getMaxInactiveInterval：获取 Session 的超时时长，默认是 1800 秒，tomcat 默认配置
  • invalidate()：让当前 Session 过期，和 Cookie 不同，无法通过设置 0 来删除

注意：通过方法名可以看出来 Inactive Inerval 指的是 Session 的 不活跃时间，如果设置时间为 3 秒，那么只要该 Session 每 2 秒活跃一次，就会一直刷新该时间，让 Session 永远不会过期，必须当 Session 不活跃时间 超过 3 秒才会被删除

Session 与 Cookie

  1. 初次发送请求时，无法获取 Session，所以会创建 Session，然后将 ID 通过 Cookie 的形式传递给浏览器 Set-Cookie: JSESSIONID=9672DA92213FAAD189983CBF25AEE41D; Path=/servlet_study; HttpOnly，这里的 JSESSIONID 就是 Session 的 ID
  2. 之后的每次请求，都会携带 Cookie: JSESSIONID=9672DA92213FAAD189983CBF25AEE41D; 该 Cookie 发送至服务端，然后 request.getSession() 便可以通过 Session ID 来获取之前创建的 Session
  3. 如果此时在浏览器端删除了该 Cookie，那么就会丢失 Session ID，下次发送请求时，就会再次创建新的 Session
