Filter

  1. Filter 过滤器是 JavaWeb 三大组件之一
  2. Filter 过滤器是一个接口
  3. Filter 过滤器作用：拦截请求，过滤响应

Filter 初体验

背景：web 目录下有一个 amdin 文件夹，其中的资源 a.html 必须要登陆后才可以访问

  1. 实现 Filter 接口
  2. 在 web.xml 中配置 Filter

Filter 的生命周期

  1. 构造器方法
  2. init 初始化方法
  3. doFilter 过滤方法
  4. destroy 销毁方法

Filter Config

FilterConfig 过滤器配置对象，用于获取 Filter 的初始化参数，以及设置 Filter 的上下文环境

Filter Chain

FilterChain 过滤器链对象，表示 Filter 的执行链条，可以通过 doFilter 方法放行请求，当存在多个 Filter 时，FilterChain 会按照配置的顺序依次执行 Filter 的 doFilter 方法

下面案例的执行结构是：Filter 1 前置代码 -> Filter 2 前置代码 -> target.jsp -> Filter 2 后置代码 -> Filter 1 后置代码，可以看出 FilterChain 的放行机制是：先执行前置代码，等待过滤器的前置代码都执行结束后，在执行目标资源代码，然后再反向执行后置代码，就像 Java 中调用对象的调用链一样

在 FilterChain 中，执行的顺序取决于 web.xml 中定义的顺序，每一个 Filter 都必须执行 doFilter 方法来放行请求，否则请求就会被拦截在当前 Filter 中，无法继续向下执行

  1. 所有过滤器以及访问的目标资源都在同一个线程中
  2. 所有过滤器以及访问的目标资源都是用的同一个 request 和 response 对象

Filter 拦截路径

Filter 的拦截路径可以通过 web.xml 中的 <url-pattern> 标签进行配置，常见的配置方式有以下几种：精确匹配、路径匹配、扩展名匹配

精确匹配

精确到某个资源，例如 /index.html，只有访问这个资源时才会被 Filter 拦截

路径匹配

路径匹配使用 /* 来表示，例如 /admin/*，表示拦截 /admin 目录下的所有资源，包括子目录中的资源

扩展名匹配

扩展名匹配使用 * 来表示，例如 *.jsp，表示拦截所有以 .jsp 结尾的资源
