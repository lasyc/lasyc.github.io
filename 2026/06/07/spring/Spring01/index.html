什么是 Spring

Spring 是一个开源的 Java 企业级应用开发框架，提供了全面的基础设施支持，帮助开发者更轻松地构建高质量、可维护和可扩展的应用程序。Spring 的核心特性包括依赖注入（DI）、面向切面编程（AOP）、数据访问、事务管理、Web 开发等。

Spring 广义和狭义的定义

广义和狭义的 Spring

  • 广义的 Spring：指整个 Spring 生态系统，包括 Spring Framework、Spring Boot、Spring Cloud 等多个子项目，涵盖了从基础设施到微服务架构的各个方面。
  • 狭义的 Spring：通常指 Spring Framework，即 Spring 的核心模块，提供了依赖注入(IOC)、面向切面编程(AOP)等基础功能，是构建 Spring 应用的基础。

Spring 的核心模块

  • IOS：Inversion of Control 的缩写，中文名称为 控制反转，是指将对象的创建和依赖关系的管理交给 Spring 容器来处理，而不是由应用程序代码来控制。这种设计模式使得代码更加松耦合、易于测试和维护。
  • AOP：Aspect-Oriented Programming 的缩写，中文名称为 面向切面编程，是一种编程范式，允许开发者将横切关注点（如日志记录、事务管理、安全性等）从业务逻辑中分离出来，以提高代码的模块化和可维护性。

Spring 的特点

  1. 非侵入式：Sprint 代码不会对工程代码造成侵入，开发者可以选择使用 Spring 的功能，而不必完全依赖于它。
  2. 控制反转：IOC 使得对象的创建和依赖关系的管理交给 Spring 容器来处理，降低了代码的耦合度。
  3. 面向切面编程：AOP 允许开发者将横切关注点从业务逻辑中分离出来，提高了代码的模块化和可维护性。
  4. 容器：IOC 就是 Spring 的核心容器，负责管理对象的生命周期和依赖关系。
  5. 组件化：Spring 提供了丰富的组件和模块，开发者可以根据需要选择使用，构建灵活的应用程序。
  6. 一站式：Spring 提供了从基础设施到微服务架构的全方位支持，帮助开发者更轻松地构建高质量、可维护和可扩展的应用程序。

Spring 模块组成

  1. Sprint Core(重点):
    • sprint-core：提供了核心功能，如依赖注入（DI）和控制反转（IOC）。
    • spring-beans：提供了 BeanFactory 和 ApplicationContext 等容器实现，用于管理
      Java 对象的生命周期和依赖关系。
    • spring-context：提供了上下文支持，如国际化、事件传播等。
    • spring-expression：提供了强大的表达式语言，用于在配置文件中动态计算值。
  2. Spring AOP：
    • spring-aop：提供了面向切面编程（AOP）的支持，允许开发者将横切关注点从业务逻辑中分离出来。
    • spring-aspects：提供了对 AspectJ 的支持，允许开发者使用 AspectJ 的语法来定义切面。
    • spring-instrument：提供了类加载器的支持，用于在运行时修改类的字节码。
  3. Spring Data Access：
    • spring-jdbc：提供了 JDBC 的支持，简化了数据库访问的代码
    • spring-orm：提供了对 ORM 框架的支持，如 Hibernate、JPA 等。
    • spring-oxm：提供了对对象-XML 映射的支持，如 JAXB、Castor 等。
    • spring-jms：提供了对 Java 消息服务（JMS）的支持，简化了消息传递的代码。
    • spring-tx：提供了事务管理的支持，允许开发者在应用程序
  4. Spring Web：
    • spring-web：提供了基础的 Web 支持，如 MultipartResolver、WebUtils 等。
    • spring-webmvc：提供了基于 Servlet 的 MVC 框架，允许开发者构建灵活的 Web 应用程序。
    • spring-websocket：提供了对 WebSocket 的支持，允许开发者构建实时通信的 Web 应用程序。
    • spring-webflux：提供了基于 Reactor 的响应式 Web 框架，适用于构建高性能、可伸缩的 Web 应用程序。
  5. Spring Message:
    • spring-messaging：提供了对消息传递的支持，如 STOMP、WebSocket 等。
  6. Spring Test：
    • spring-test：提供了对 Spring 应用程序的测试支持，如单元测试、集成测试等。

分析

  1. 给 User 添加构造方法，在加载配置文件时，构造方法会被调用。
  2. Spring 通过反射机制创建对象，会加载配置文件，并解析，通过 bean 标签中的 id 和 class 属性来创建对象，id 属性指定了对象的名称，class 属性指定了对象的类名。
  3. 创建好的对象会被保存到 beanDefinitionMap 中，beanDefinitionMap 是一个 HashMap，以 id 作为键，以 BeanDefinition 作为值，BeanDefinition 是一个接口，包含了对象的定义信息，如类名、作用域、生命周期等。
  4. 获取对象时，Spring 会根据 id 从 beanDefinitionMap 中获取 BeanDefinition，然后通过反射机制创建对象，并返回给调用者。

整合 Log4j2 日志框架

Log4j2 是 Apache 提供的一个 Java 日志框架，具有高性能、灵活性和易用性等特点。Spring 可以与 Log4j2 集成，提供更强大的日志功能。

  • 日志优先级：FATAL > ERROR > WARN > INFO > DEBUG > TRACE，日志级别越高，输出的日志信息越少。
  • 日志输出方式：ConsoleAppender（控制台输出）、FileAppender（文件输出）、RollingFileAppender（滚动文件输出）等。
  • 日志格式：可以通过 PatternLayout 来定义日志的输出格式，如 %d{yyyy-MM-dd HH:mm:ss} [%t] %-5p %c{1} - %m%n，表示输出日志的时间、线程名称、日志级别、类名和日志信息。
