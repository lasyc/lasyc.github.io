事务

事务：事务是一组操作的集合，该集合中的所有操作被视为一个整体，要么全部执行成功，要么全部执行失败。若任意一个操作执行失败，则事务回滚，所有操作全部撤销。

事务四大特性 ACID

  1. 原子性（Atomicity）：事务是一个不可分割的工作单位，事务中的操作要么全部完成，要么全部不完成，不会只完成一部分。
  2. 一致性（Consistency）：事务必须使数据库从一个一致性状态变换到另一个一致性状态。一致性状态：指事务执行前后数据库中数据的完整性和正确性。
  3. 隔离性（Isolation）：一个事务的执行不能被其他事务干扰，即一个事务内部的操作及使用的数据，对并发的其他事务是隔离的，并发执行的各个事务之间不能互相干扰。
  4. 持久性（Durability）：事务执行完成之后，对数据库中数据的改变就是永久的，即数据改变之后，其他事务可以访问这些数据。

@Transactional

@Transactional 注解是 Spring 框架中用于声明式事务管理的重要工具。它可以应用于类或方法，指示 Spring 在调用该方法时启动一个新的事务，或在现有事务中执行操作。

isolation

isolation 用于设置事务的隔离级别，详情查看 MySQL 篇

值 隔离级别 问题
Isolation.DEFAULT(-1) 默认 使用数据库默认的隔离级别，MySQL 默认为可重复读
Isolation.READ_UNCOMMITTED(1) 读未提交 会发生脏读、不可重复读、幻读
Isolation.READ_COMMITTED(2) 读已提交 解决脏读问题，但会发生不可重复读、幻读
Isolation.REPEATABLE_READ(4) 可重复读 解决脏读、不可重复读问题，但会发生幻读
Isolation.SERIALIZABLE(8) 串行化 解决全部问题，不会发生任何问题，但效率最低

propagation

propagation 用于控制事务的传播行为，当两个 service 方法都含有事务，那么当两个方法之间调用时，需要设置事务如何传递，用你的还是用我的？还是合并？还是再开一个新的事务？

传播行为 描述
REQUIRED 有就加入，没有就新建
SUPPORTS 有就加入，没有就没有
MANDATORY 有就加入，没有就抛异常
REQUIRES_NEW 不管有没有，开一个新的事务，之前的事务被挂起
NOT_SUPPORTED 不使用事务，如果存在事务，就把事务挂起
NEVER 不使用事务，如果存在事务，就抛异常
NESTED 如果有事务，就在当前事物中开启一个子事务，子事务独立于外部事物
如果没有事务，和 REQUIRED 一样
