i18n 国际化

发表于2026-07-18|更新于2026-07-18
|浏览量:

I18N

I18N 是 Internationalization 的缩写，中文名称为 国际化，是指在软件开发过程中，使软件能够适应不同语言、文化和地区的需求，以便在全球范围内使用。

Java I18N

Java 自身就支持 I18N，前边在学习 Java Web 的时候学习过，Java Web I18N

  • 创建资源文件，按照 前缀_语言_国家 的格式命名
  • 使用 ResourceBundle 和 Local 获取资源文件内容

注意：如果中文的资源文件读取后是乱码，因为 IDEA 中 properties 文件默认采用 ISO-8859-1 编码格式，而 ResourceBundle 读取值时使用 UTF-8，需要修改文件编码格式

Spring I18N

在 Spring 中，提供了 MessageSource 接口，并且提供了四个实现类

实现类 数据来源 是否可热加载 适用场景
ResourceBundleMessageSource .properties 文件 否（缓存） 大多数项目首选
ReloadableResourceBundleMessageSource .properties 文件 是 需要运行时更新翻译
StaticMessageSource 代码注册 否 单元测试
DelegatingMessageSource 委托 parent - Spring 内部使用

ResourceBundleMessageSource

ResourceBundleMessageSource 是基于 Java 的 ResourceBundle 实现的，也是 Spring 默认使用的

  • AbstractApplicationContext 抽象类的 initMessageSource 方法源码
  • 因为 ApplicationContext 继承了 MessageSource，所以直接使用 context 调用 getMessage 方法
