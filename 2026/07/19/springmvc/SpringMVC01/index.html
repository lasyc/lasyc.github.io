什么是 MVC

MVC 是一种软件架构思想，将软件按照 Model（模型）、View（视图）、Controller（控制器）来进行划分

  • Model：模型层，指工程中的 JavaBean，其中包括了实体类 Bean，比如：Person、User 等，还有业务 Bean，比如：UserController、UserService 等
  • View：视图层，指工程中的前端交互页面，比如：html、jsp 等
  • Controller：控制层，指工程中的 Servlet（参考JavaWeb），用于接受请求和响应

MVC 的工作流程：

用户通过 View（视图层）交互，发送请求到 Controller（控制层），Controller（控制层）接收请求开始处理，调用 Model 层的业务 Bean 处理数据，将数据保存在实体类 Bean 中，然后通过响应返回给 View（视图层）来展示数据或者处理结果

什么是 Spring MVC

Spring MVC 是 Spring 的一个后续产品，是 Spring 的一个子项目，用于解决 Spring 为表述层开发提供的一套完整的解决方案

Spring MVC 的特点

  • Spring 家族产品，与 Spring 其他组件可以无缝组合
  • 基于原生 Servlet，通过前端控制器 DispatcherServlet 对请求和响应进行统一处理
  • 代码清新简洁
  • 组件化程度高
  • 性能卓越
