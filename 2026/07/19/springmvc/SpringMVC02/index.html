创建 maven 工程

  1. 添加 web 模块

  2. 打包方式：war

  3. 添加依赖，建议加载父工程

配置 web.xml

前边提到过，Spring MVC 基于原生 Servlet，通过前端控制器 DispatcherServlet 对请求和响应进行统一处理，因此需要在 web.xml 文件中对 DispatcherServlet 进行配置

  • 配置 DispatcherServlet，不指定 Spring MVC 的配置文件路径，默认在 WEB-INF 下，名为 [servlet-name]-servlet.xml，也就是 DispatcherServlet-servlet.xml

  • 指定 Spring MVC 的配置文件路径

  • 在 resources 下创建 Spring MVC 配置文件

创建请求控制器

DispatcherServlet 对前端的请求和响应做了统一的处理，因此我们只需要创建控制器类（Controller）即可，其变学习 Spring 的时候也提到了，可以使用 @Controller 注解将对象交给 Spring IOC 容器进行管理

注解 作用
@Controller Controller 类，控制器层
@Service Service 类，业务层
@Repository Mapper 类，持久层
@Component 普通 Bean

  • 创建控制器类

创建 Spring MVC 的配置文件

  • 在配置文件中开启组件扫描

  • 根据配置文件中的前缀和后缀规则，在 WEB-INF 文件夹下创建 templates 文件夹，并创建 HTML 文件

测试 Hello World

  • 实现对首页的访问

因为 @RequestMapping 配置的是根目录，所以打开项目就是对应的请求路径，所以可以直接访问首页

访问其他页面

  • 在首页中添加连接，使用 thymeleaf 语法发送请求
  • 创建目标页面
  • 创建对应请求路径的方法

总结

浏览器发出请求，DispatcherServlet 在接收请求后会去查找与之匹配的路径，控制器类需要使用 @Controller 标记，并且其中的方法需要使用 @RequestMapping(value = "url") 绑定，然后执行对应的方法，该方法的返回值可以是字符串，默认将会该字符串识别为视图名，通过配置文件中的前缀和后缀拼接成完整的路径，访问对应的资源，然后转发视图到页面
