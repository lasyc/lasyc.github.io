RequestMapping 注解

发表于2026-07-20|更新于2026-07-20

@RequestMapping 注解的功能

@RequestMapping 注解的主要功能就是将请求路径和方法进行绑定，根据请求路径执行对应的方法

@RequestMapping 注解的位置

@RequestMapping 可以写在类和方法上，如果添加在类上，则作为该类中所有方法的前置路径，会将该路径与方法上的路径进行拼接，形成完整路径

注意：在全部的控制器中，请求路径不可以重复

假设我们有两个控制器，一个用于处理“老师”相关的请求，另一个负责“学生”相关的请求，现在有两个方法，分别用于获取老师信息和学生信息

可以看到两个方法的请求路径重复了，此时我们就可以在类上在添加一个 @RequestMapping 注解用于区分

@RequestMapping 中的属性

@RequestMapping 中的属性：

属性名 值 描述
name 名称 给这个映射起一个逻辑名称
value 请求路径 指定映射的 URL 路径模式
path 同 value 同 value
method 请求方法 限定只接受哪些 HTTP 方法。不指定则所有方法都匹配。
params 请求参数 根据 URL 的查询参数（?key=value）做精细匹配：
headers 请求头 根据 HTTP 请求头 做匹配，语法与 params 相同
consumes 请求体 限定客户端发送的 Content-Type。只有请求体的类型匹配时才会进入该方法。
produces 响应体 限定该方法产出的响应 Content-Type，同时也会根据客户端的 Accept 头做内容协商。
version 版本 用于将请求路由到特定版本的接口实现，常配合自定义的版本解析器（从请求头、URL路径、参数中提取版本号）。

name

name — 昽射名称

给这个映射起一个逻辑名称，主要用于：

  • 日志/调试时识别映射
  • 配合 MvcUriComponentsBuilder 做命名式URL构建
  • 测试中按名称查找映射

value 和 path

value / path — 请求路径（互为别名）

value / path 属性可以接收数组作为参数，也就是说可以设置多个请求路径到一个方法上，匹配任意一个均可

指定映射的 URL 路径模式，支持：

  • ? : 匹配单个字符：/user? 匹配 /user1
  • * : 匹配任意路径片段：/user/* 匹配 /user/1
  • ** : 匹配多层路径：/user/** 匹配 /user/1/2/3
  • {id} : 路径变量：/user/{id} 可通过 @PathVariable 提取
